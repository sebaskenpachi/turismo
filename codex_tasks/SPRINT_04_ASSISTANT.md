# Tarea Codex — Sprint 4: asistente turístico inteligente básico

Mejora el asistente actual para recomendar rutas según contexto.

Objetivo:
El usuario debe poder recibir una recomendación útil según tiempo, presupuesto, movilidad e intereses.

Entradas:
- Tiempo disponible en minutos.
- Presupuesto: bajo, medio, alto.
- Movilidad: caminando, auto, taxi/uber.
- Interés: naturaleza, historia, comida, familiar, pareja, vida nocturna.
- Con niños: sí/no.

Reglas iniciales:
- Poco tiempo + caminando = Cinta Costera o parte corta de Casco Viejo.
- Naturaleza = Cerro Ancón.
- Historia = Casco Viejo.
- Bajo presupuesto = rutas caminables y comida local.
- Con niños = evitar caminatas difíciles y priorizar rutas fáciles.
- Pareja/noche = Casco Viejo y rooftops.

Entregable:
- UI simple para capturar preferencias.
- Recomendación clara tipo concierge.
- Explicación de por qué recomendó esa ruta.
