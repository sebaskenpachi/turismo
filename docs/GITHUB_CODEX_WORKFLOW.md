# Flujo GitHub + Codex

## Objetivo
Que Sebastián no programe. Codex ejecuta tareas y Sebastián valida como QA de usuario.

## Flujo operativo
1. Crear repositorio privado en GitHub: `rutas-panama`.
2. Subir el proyecto base.
3. Abrir Codex en el repositorio.
4. Copiar una tarea desde `codex_tasks/`.
5. Pedirle a Codex que haga cambios pequeños.
6. Revisar resumen y ejecutar pruebas.
7. Generar APK.
8. Sebastián prueba en teléfono.
9. Reportar feedback.
10. Repetir.

## Reglas
- Nunca pedir “haz toda la app”.
- Cada sprint debe tener máximo una gran función.
- No aceptar cambios sin resumen.
- No aceptar llaves quemadas en código.
- No avanzar a Google Places si el mapa/GPS no funciona.
- No avanzar a monetización si la experiencia básica no se siente útil.

## Primer prompt a usar
Usar `codex_tasks/SPRINT_00_STABILIZE.md`.
